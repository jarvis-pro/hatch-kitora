# Getting Started

Kitora 的入门资料按"阶段 + 读者背景"组织。**建议阅读顺序**：先读跟你栈最接近的速通对照（让你能动手），再按需读体系化学习路线（打基础），最后读模块深挖（成为模块负责人）。

## 现有资料

### 速通对照（按读者背景）

| 文档                                   | 适合读者                                  | 阅读时长     |
| -------------------------------------- | ----------------------------------------- | ------------ |
| [vue-to-nextjs.md](./vue-to-nextjs.md) | 熟练 Vue 3 + Node.js (Midway.js) 的工程师 | 约 60–90 min |

### 通用进阶

| 文档                                         | 适合阶段                             | 投入时长             |
| -------------------------------------------- | ------------------------------------ | -------------------- |
| [learning-path.md](./learning-path.md)       | 速通过后的 4–6 周体系化学习路线      | 每周 15–20 h         |
| [module-deep-dive.md](./module-deep-dive.md) | 资深工程师向的模块设计权衡与边界情况 | 约 3–4 h（按需查阅） |

## 维护约定

- 每份**速通对照**文档都应包含：心智模型映射 → 必学清单 → 模块设计要点 → 第一周路线图 → 反模式速查 这五个部分，便于读者横向对照不同入门路径。
- 项目大版本（`package.json` `version` 升 minor）时统一回顾一次，刷新被新增/重构的模块。
- 如果你正在写**新的速通对照**（比如"给 Java/Spring 同学的"、"给 PM/运营的产品概念入门"），命名遵循 `<from-stack>-to-<our-stack>.md` 或 `<role>-overview.md`，并在上面"速通对照"表格登记一行。
- `learning-path.md` 和 `module-deep-dive.md` 是**通用文档**，所有读者共享——内容随项目演进迭代，不按读者背景分版本。
